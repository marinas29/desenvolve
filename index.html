Git e Github

O que é o GitHub?

O GitHub é uma plataforma onde você pode armazenar seus projetos. É como se fosse uma rede social, só que de códigos, onde seus desenvolvedores podem disponibilizá-los para outras pessoas verem.

O que é o Git?

O Git é um sistema de controle de versão de arquivos. É um software livre e muito utilizado no desenvolvimento de software onde diversas pessoas estão contribuindo simultaneamente, podendo criar e editar arquivos. Sempre quando alguém disponibiliza sua parte do projeto no Git, ele gerencia as alterações feitas e guarda um histórico. Isso é importante pois se houver algum problema você pode desfazer as alterações e voltar para a versão que estava estável.

Comando ADD e COMMIT

Comando para adicionar qualquer arquivo alterado no índice, ou seja, adiciona como uma das alterações a serem armazenadas no commit.

git add nome_do_arquivo

Depois de terminar as alterações, você faz o commit, dessa forma ajuda a organizar sua evolução do código e organizar por tópicos (commits)

git commit -m “Adicione qualquer mensagem sobre o commit aqui”

Seu repositório foi clonado antes de começar o desenvolvimento?

Se sim, na hora de enviar as alterações para o repositório remoto (Github), basta seguir com o comando:

git push remote branch

Não foi vinculado ao Github ainda?

Se, no entanto, um repositório existente ainda não tiver sido clonado e você pretende estabelecer uma ligação entre o seu repositório e um servidor remoto, execute o seguinte comando:

git remote add origin servidor (Substitua servidor pelo endereço do servidor remoto.)

Branches

Outra característica brilhante (mas avançada) do GIT é sua capacidade de permitir que desenvolvedores e gerentes de projeto criem vários ramos (branches) independentes dentro de um único projeto.

Comando para CRIAR uma Branch: git checkout -b nome_branch

Comando para ir para outra branch: git checkout nome_branch

Fiz todas as alterações e quero disponibilizar no GitHub

git push origin nome_branch

Casos comuns:

Houve uma alteração no Github e no meu repositório local (computador) não está atualizado.

Nesse caso, utilizamos o comando git pull, ele faz com que a BRANCH local pegue todas as alterações que estão no repositório remoto.

Exemplo de um merge:

Nessa semana recebi uma tarefa que precisava criar um carrinho para o projeto

Então eu fui até a main e a partir dela criei uma branch chamada feature-cart.

Depois de uma semana, terminei meu desenvolvimento mas percebi que entraram outras correções na main, e tenho um problema porque a minha branch está desatualizada.

Nesse caso vamos precisar utilizar o comando git merge

Para isso, esteja na branch desatualizada (no exemplo é a feature-cart) e adicione ao final do comando a branch que quer pegar as novas alterações, exemplo: git merge main, irá trazer todas as alterações da main para a minha branch atual
